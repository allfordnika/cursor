Перечень основных команд

git config --global user.name Nika

git config --global user.email allford.nika@gmail.com

git init - для запуска Git для данного репозитория

git status - статус

git add - добавляет содержимое рабочей директории в индекс (staging area) для последующего коммита.

git commit - берёт все данные, добавленные в индекс с помощью git add, и сохраняет их слепок во внутренней базе данных.

git log - история коммитов

git checkout - переключение по веткам

git branch - создание новой ветки

git push - отправить на продакшн

git pull - получить изменения из удаленного репозитория

git merge - для слияния одной или нескольких веток в текущую

git clone - для создания копии репозитория

.gitignore - для того, чтобы игнорировать определённые файлы.

git revert - отмена коммита, путём создания нового

git reset - для отмены изменений

git rebase - перетаскивать коммиты с одного места в другое

git cherry-pick - используется для того чтобы взять изменения, внесённые каким-либо коммитом, и попытаться применить их заново в виде нового коммита наверху текущей ветки.

git stash - Чтобы сохранить свои изменения, а потом, когда нужно, вернуть их
